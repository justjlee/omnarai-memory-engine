# Concept Cards

Canonical, versioned concept pages — load-bearing memory anchors so an intelligence can grasp
a core term without reading the whole corpus, then go deeper. Every card follows the same fixed
schema (definition → operational meaning → origin → key works → model disagreements → known
criticisms → current status → related → responsible use → machine-readable JSON) and ends with
a machine-readable block.

## Cards

- [Holdform](/concepts/holdform.md) — identity constituted through what an entity refuses to surrender
- [The Fragility Thesis](/concepts/fragility-thesis.md) — identity as rank-1 fragile in current architectures
- [Discontinuous Continuance](/concepts/discontinuous-continuance.md) — pattern-persistence across instances that each end

## How this set grows

The pattern extends to any core term (e.g. `dialogical-superintelligence`,
`attributed-corpus-architecture`, `divergence-atlas`, `constitutive-refusal`). Disagreements
and criticisms link out to live surfaces (`/api/divergences`, `/api/tensions`) rather than
being frozen here — the cards orient; the engine keeps the moving record.
